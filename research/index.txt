# 7
Nama Uji:
Logout dan expiration membatalkan Session token, sehingga tombol back tidak dapat menggunakan Session token otentikasi sebelumnya.

Keterangan:
Aplikasi tidak menampilkan halaman yang membutuhkan autentikasi saat setelah logout dan tekan tombol back. Percobaan refresh sudah dilakukan dan di redirect ke halaman login

Hasil:
Saat user logout, terkadang ketika back ke page lain suka ada page yang nyangkut karena browsernya belum refresh atau page lain masih menyimpan token. Untuk itu seharusnya ada validasi middleware tambahan jika diperlukan atau second layer guard untuk ngehandle itu, alhasil ketika sudah logout dan di back atau pindah page lain maka akan otomatis di cek dan di redirect ke halaman login

# 8
Nama Uji:
Jika autentikator mengizinkan  (user) untuk tetap masuk, pastikan bahwa autentikasi ulang terjadi secara berkala baik saat digunakan secara aktif atau setelah periode idle. (15 menit tidak aktif, dengan 2FA)

Keterangan:
Tidak ada tombol tetap masuk. Aplikasi mengizinkan user untuk tetap login selama 15 menit. Apabila idle timeout maka akan otomatis ternotif dan diminta untuk login ulang

Hasil:
Untuk ini nantinya idle akan dideteksi oleh fe dan be, namun untuk yang tugasnya hapus token atau reset token nantiya dari be, fe nantinya akan menampilkan modal/drawer pemberitahuan, nantinya akan otomatis di redirect untuk login lagi ketika di close/action modal/dialognya. Nantinya fe bisa bikin hooks atau pake library hooks tertentu untuk deteksi aktivitas user jika idle.

# 9
Nama Uji:
Aplikasi memberikan opsi untuk menghentikan semua session aktif setelah perubahan kata sandi yang berhasil (termasuk perubahan melalui pengaturan ulang/pemulihan kata sandi), dan bahwa ini efektif di seluruh aplikasi, login gabungan (jika ada).

Keterangan:
Saat proses penggantian password dengan fitur reset password, pada saat pertama kali user melakukan login dengan password baru, maka sesi lama akan berakhir secara otomatis

Hasil:
Ini juga mostly di handle di backend, nantinya fe tinggal handle apabila session cookie udah gaada, alhasil error saat fetch data, dan disitu harus handle ke halaman login lagi

# 10
Nama Uji: 
Pastikan pengguna (user) dapat melihat dan setelah memasukkan kembali kredensial, pengguna (user) keluar dari salah satu atau semua session dari perangkat yang sedang aktif

Keterangan: 
Aplikasi saat ini menerapkan maksimal 1 sesi untuk 1 user. Apabila pada saat bersamaan ada username yang login di perangkat lain, maka sesi lama akan ter-logout.

Hasil:
Ini nanti yang simpan, generate, set, validasi dan baca cookies dari backend, frontend nantinya handle logout aja harusnya, dan pastiin ketika logout semua data cookies nya itu hiang dari browser, kemudian ketika di back atau ke page lain yang butuh credentials, app tersebut langsung di redirect ke login dan tidak muncul apa2
